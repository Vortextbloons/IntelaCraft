# Voxel Renderer

`services/voxel-renderer` is a dependency-free Go child process managed by the controller. It accepts versioned newline-delimited JSON requests on stdin and writes one correlated JSON response per line on stdout. Supported commands are `health`, `process_scene`, `render_image`, `generate_thumbnail`, and `compare_snapshots`.

Jobs are capped at 32,768 blocks and 2048×2048 output. Scene processing removes fully hidden blocks and reports visible face directions. Static rendering produces deterministic, bounds-fitted isometric PNG files at controller-allocated paths; renderer input cannot select paths directly through the public API. Representative material families provide stable Minecraft-like colors and face shading for stone, wood, earth, grass, sand, brick, metal, glass, foliage, water, lights, and verification overlays. The controller applies request timeouts, rejects failed jobs, restarts the process on a subsequent request after exit, and removes temporary render files older than one hour at startup.

The interactive webview renderer uses the same material-family behavior with procedural 16×16 face textures. It groups blocks by type into instanced meshes, applies sRGB color management and filmic tone mapping, supports transparent and emissive families, and fits the camera to the visible build. Procedural textures intentionally avoid bundling or redistributing Minecraft resource-pack assets.
